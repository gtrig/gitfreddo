# Internationalization (i18n)

GitFreddo supports **English** (`en`) and **Greek** (`el`) for UI chrome. Git output and user content are not translated.

## Stack

- [react-i18next](https://react.i18next.com/) + [i18next](https://www.i18next.com/)
- Locale files: `src/locales/en.json`, `src/locales/el.json`
- User preference: **Settings → Interface → Language**

## Key naming

Use dot-separated namespaces:

```
workspace.hub.openFolder
actions.push
settings.language.label
toast.pushSuccess
errors.pushFailed
```

Group keys by feature area, not by file name.

## Adding strings

1. Add the English key to `src/locales/en.json`
2. Add the Greek translation to `src/locales/el.json`
3. Replace the literal in the component:

```tsx
import { useTranslation } from 'react-i18next'

function MyComponent() {
  const { t } = useTranslation()
  return <button>{t('actions.push')}</button>
}
```

### Interpolation

```json
{
  "branch": {
    "deleteConfirm": "Delete branch \"{{name}}\"?"
  }
}
```

```tsx
t('branch.deleteConfirm', { name: branchName })
```

## What to translate

| Translate | Do not translate |
|-----------|------------------|
| Button labels, menu items | Branch names, remote URLs |
| Dialog titles and descriptions | Commit messages, git errors (raw) |
| Toast messages (known cases) | SHAs, file paths from git |
| Settings labels | Porcelain/git command output |
| Empty states and placeholders | AI-generated content |

## Context menus

Label builders in `src/lib/*ContextMenu.ts` accept a `t` function parameter instead of hardcoded strings:

```ts
export function buildCommitContextMenu(t: TFunction, ...) {
  return [{ label: t('contextMenu.reword'), ... }]
}
```

## Date and time

`formatTimeSince()` accepts a locale for `Intl.RelativeTimeFormat`.

## Greek translation guidelines

- Use standard Greek UI terminology; git terms often stay transliterated or in English where conventional (e.g. "commit" → "commit" or "υποβολή" — be consistent per namespace)
- Keep strings concise — Greek text can be longer; verify layout in the UI
- Maintain formal tone (σελίδα settings, not slang)

## Documentation

User guides in `docs/` are **English only** for this milestone. App UI locale is independent of doc language.

## Regression guard

Optional script `scripts/check-i18n.sh` flags new hardcoded JSX text in migrated directories during CI.
