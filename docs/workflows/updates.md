# Application updates

GitFreddo uses [electron-updater](https://www.electron.build/auto-update) with **GitHub Releases** as the update feed.

## How it works

1. On startup (if enabled in **Settings → Maintenance**), the app checks the latest release on GitHub after a short delay.
2. When a newer version is found, a banner offers **Download** and later **Restart to update**.
3. **Help → Check for Updates** runs a manual check and shows a toast if you are already up to date.

Packaged builds include `latest.yml` / `latest-mac.yml` / `latest-linux.yml` metadata generated by electron-builder. These files are attached to each GitHub Release by CI.

## Settings

| Setting | Effect |
|---------|--------|
| Update channel | **Stable** — release builds only; **Beta** — includes prereleases |
| Check on startup | Delayed background check when the app opens |
| Auto-download | Download updates in the background when available |

## Development

Auto-update is **disabled** when running unpackaged (`npm run dev`). Manual checks report the current dev version as up to date.

## Platform notes

| OS | Installer | Auto-update |
|----|-----------|-------------|
| Windows | NSIS `.exe` | Supported |
| macOS | DMG + ZIP | ZIP used for updates; code signing recommended |
| Linux | AppImage | Supported; `.deb` users should update manually |

## Release requirements

Tagged releases (`v*`) bump `package.json` to the tag version before building, then upload installer artifacts **and** `*.yml` update metadata. See `.github/workflows/release.yml`.

### Preparing a release locally

1. Run `npm run release:prepare -- vX.Y.Z` to sync `package.json` / `package-lock.json` with the tag.
2. Move `[Unreleased]` notes in `CHANGELOG.md` and `NEWS.md` into `## [X.Y.Z]`.
3. Commit, create the tag, and push branch + tag.

The repo installs a **pre-push** hook (via `npm install` → `prepare`) that blocks pushing a `v*` tag unless `package.json` already matches that semver. Use `--no-verify` only when you know what you are doing.

Forge OAuth for installed apps is baked at build time from Actions secrets:

| Secret | Maps to build env |
|--------|-------------------|
| `GITFREDDO_GITHUB_CLIENT_ID` | `GITHUB_CLIENT_ID` (Actions forbids secrets named `GITHUB_*`) |
| `BITBUCKET_CLIENT_ID` | `BITBUCKET_CLIENT_ID` |
| `BITBUCKET_CLIENT_SECRET` | `BITBUCKET_CLIENT_SECRET` |
